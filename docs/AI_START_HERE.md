# AI 開發入口

開始任何 Sprint 前，必須先閱讀專案根目錄的 `README.md`，再依序閱讀 `docs/01_專案概觀.md` 至 `docs/11_AI題庫維護規範.md`。在完成閱讀前，不得直接開始修改程式。

開發原則：

- 最小修改，不重構既有架構。
- 不建立重複頁面。
- 不新增假資料。
- Build 成功後才可 Deploy。
- Sprint 完成即停止，不自行開始下一個 Sprint。

每次 Sprint 完成後，必須同步更新：

- `docs/05_功能進度.md`
- `docs/06_Sprint紀錄.md`
- `docs/09_Technical_Debt.md`
- `docs/10_Roadmap.md`
