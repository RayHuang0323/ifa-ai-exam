# AI 開發入口

開始任何 Sprint 前，必須先閱讀專案根目錄的 `README.md`、`docs/00_PROJECT_STATE.md` 與 `docs/13_Codex驗收與Debug規範.md`，再依任務查閱其他 Wiki。開發、修 Bug 或完成 Sprint 後，必須閱讀並遵守 `docs/13_Codex驗收與Debug規範.md`。在完成閱讀前，不得直接開始修改程式。

開發原則：

- 最小修改，不重構既有架構。
- 不建立重複頁面。
- 不新增假資料。
- Build 成功後才可 Deploy。
- Sprint 完成即停止，不自行開始下一個 Sprint。

每次 Sprint 完成後，必須同步更新：

- `docs/05_功能進度.md`
- `docs/06_Sprint紀錄.md`
- `docs/09_Technical_Debt.md`
- `docs/10_Roadmap.md`
